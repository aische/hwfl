# Decision log ‚Äî 2026-07

## 2026-07-15 ‚Äî Greenfield language direction

- Restart computation substrate as a **minimal ML** embedded in markdown
  modules; keep hwfi as a side-by-side reference, not the forever host of
  new features.
- Durable resume stays at **host-op / effect boundaries**, not every pure
  reduction (same transition idea as hwfi v2 machine).
- Default LLM library remains **llm-simple**, but workflows must not depend
  on it: introduce an internal **`LlmProvider`** adapter from day one.
- Spec lives as a multi-file pack under `spec/`; project workflow mirrors
  hwfi (`STATUS` / `TASKS` / monthly log / Cursor handoff).
- Provisional product name: **pml**.

## 2026-07-15 ‚Äî M0 syntax skeleton landed

- Cabal package `pml` (GHC2021): kernel AST/parser/pretty + markdown loader.
- Surface choices locked for v0 parse: sequential `let` without `in`,
  record field shorthand `{ name }`, host paths via `.` projection
  (`llm.chat`), module paths via `/` qnames; `null` literal not in kernel
  (Option/Result preferred per language spec).
- `LlmProvider` / llm-simple deferred to M4 (acceptance), not wired in M0.

## 2026-07-15 ‚Äî M1 pure evaluator

- Big-step recursive pure eval (`Pml.Eval.Pure`), not an explicit CEK loop:
  matches runtime ¬ß2‚Äì3 (no snapshot mid pure reduction). Frame/CEK driver
  deferred until host transitions need resumable konts (M4+).
- Infix `+ - * /`, comparisons, `&&`/`||` elaborate in the parser to
  `EApp (EVar op) [‚Ä¶]`; op idents are prelude `VBuiltin`s (not host ops).
- Top-level `fun` decls knot-tied into one shared env (mutual call OK).
- `par` / `join` / `confirm` / `try` / `@section` / bare qnames ‚Üí
  `Unsupported` (trap if reached). No HostRef / Bytes yet.
- Arithmetic same-sort only (Int+Int or Float+Float); Int `/` is `div`.

## 2026-07-15 ‚Äî M2 type checker

- Bidirectional check (`Pml.Check.Infer`) over resolved aliases; effects on
  arrows are stripped (lattice deferred to M3).
- Module frontmatter inputs/outputs elaborate missing `main` annotations and
  are compared to the inferred `main` type.
- Host op *types* for `fs.read` / `llm.chat` live in check prelude so bodies
  like summarise type-check before M4 runtime; no effect enforcement yet.
- `schema(T)` is a parse form (`ESchema`); check-time JSON Schema for
  records/lists/bases; eval traps (compile-time operator).
- Prelude `+`/`==`/ord typed as Int for v0; Float polymorphism later.
- Record equality is by field name (order-insensitive); no width subtyping.

## 2026-07-15 ‚Äî M3 effects + check CLI

- Residual effect inference (`Pml.Check.Effects`) over apps of `TEffFun`
  host stubs plus `par`/`join`/`confirm`; module `effects:` is a hard ceiling
  (absent ‚áí pure; inferred ‚äà declared ‚áí `EffectsNotAllowed`).
- `pml check <module.md>` loads + type-checks + effect-checks one file; exit 1
  on failure, exit 2 on usage. Project.json + import graph deferred (same
  command later) ‚Äî logged on stderr in single-module mode.
- E12 fixture fails closed; summarise with `[Read, Net]` passes. No M4 host
  runtime / provider yet.

## 2026-07-15 ‚Äî M4 host + LlmProvider + snapshots

- Runtime host ops mirror check prelude only (`fs.read`/`fs.write`/`llm.chat`);
  no second API surface. Pure eval still traps on `VHostOp`.
- Workspace sandbox: lexical resolve + canonicalize prefix (symlink escape hard
  fail). FileRef at runtime is a path string under the workspace.
- `LlmProvider` record in `Pml.Llm.Provider`; engine types in `Pml.Llm.Types`.
  Default adapter `Pml.Llm.Simple` wraps llm-simple (`^>=0.1.0.1`); retries stay
  in the provider. Mock provider for CI; `--llm-provider mock|simple` and
  `PML_LLM_PROVIDER`. Workflows never import llm-simple.
- Snapshots: after each completed host op write format-1 boundary record under
  `.pml/runs/<run-id>/snapshot.json` (+ `transitions.jsonl`). Full kont /
  `machine_json` for `pml resume` deferred to M5 (logged; M4 establishes the
  host-boundary resume points only).
- `pml run <module.md> [--workspace] [--input k=v‚Ä¶]`: check then execute `main`;
  human logs on stderr; exit ‚âÝ0 on failure. Project graph still single-module.
- E03/E04 mock-green; sandbox escape rejected; 66 tests. Real network LLM not
  required in CI.

## 2026-07-15 ‚Äî M5 concurrency + human + resume

- Runtime switched to an explicit machine (`Pml.Runtime.Machine`): frames +
  `Current`, big-step pure crunch, small-step host/par/confirm transitions.
  Top-level funs are `VTopFun` so snapshots stay finite (no knot-tied env).
- `par(max)` scheduler with ordered slots; confirm inside a branch drains
  in-flight work then pauses (`awaiting_confirm`); completed slots not re-run.
- `confirm` sugar and `human.confirm` host op share the same pause gate;
  `pml approve --yes|--no` injects `Bool` and continues.
- Snapshots include full `machine_json` (+ `meta.json` entry path / hash).
  CLI: `step` / `resume` / `approve`; `pml run --step`; exit 3 pause, exit 4 stale.
- E07 ordered par, confirm+approve, par-confirm freeze, step/resume, stale hash
  covered; 71 tests. OS-thread parallelism deferred (cooperative pool is M5).

## 2026-07-15 ‚Äî M6 observability + show

- `Pml.Obs.{Span,Trace,Redact,Show}`: append-only `spans.jsonl` open/close lines
  (O(1) per transition; tree built only by `pml show`, never on the step hot path).
- Every host op + module `main` emit nested spans; `human.confirm` keeps a span
  open across pause and closes on approve. `obs.log` / `obs.span` wired (region
  via `FrRegion` / `CurCloseRegion`). Check-time `obs.span` return type is still
  Unit‚ÜíUnit (polymorphism deferred).
- Redaction: `VSecret` never cleartext in snapshots; host attrs store path/model
  and string lengths (not prompt bodies); sensitive JSON keys scrubbed on write.
- Snapshot persists `span_stack` + `span_counter` so resume keeps nesting.
- CLI: `pml show <ws> <run-id> [--tree|--spans|--snapshot] [--filter PREFIX]`.
- 74 tests (redaction + summarise-style span tree).

## 2026-07-15 ‚Äî M7 agent loop + typed tools

- `llm.agent` is a multi-transition machine current (`CurAgent`), not one opaque
  host call: model round ‚Üí sequential tool nested machines ‚Üí next model / final
  `{ text, rounds }`. Each model/tool boundary snapshots (resume mid-tool OK).
- Prelude `tool(f)` advertises host ops (`fs.read`/`fs.write`) or typed module
  funs/closures with JSON Schema from param annotations; Infer special-cases
  `tool` so any function type ‚Üí `ToolSpec`.
- `LlmProvider` `ChatRequest` gains tools + turns; adapters map to llm-simple
  `ToolDef` / content blocks. Agent rounds emit `agent_round` child spans under
  the open `llm.agent` host span.
- No llm-simple agent loop in-engine ‚Äî pml owns stepping (same idea as hwfi
  MachineAgent). Skills / `llm.agent-object` submit tool deferred.
- E15 green; 76 tests.
